# AI Assistant — Project Index

## Files

| File | Purpose |
|---|---|
| `FEATURES.md` | What every feature does and why — business logic, rules, flows. Always up to date. Read this first. |
| `DESIGN_SPECS.md` | Detailed planning snapshot — deep design decisions, tables, step-by-step flows. Reference only, may go stale on implementation details. |
| `TASKS.md` | Build progress — what's done, what's next, how deep to go per task. |
| `CODEBASE.md` | Every file and folder in the codebase explained. Updated as code grows. |
| `SETUP.md` | Step-by-step infrastructure setup guide. Reusable across projects. |
| `JOURNEY.md` | Full project lifecycle story — every decision, problem, pivot, and lesson from idea to production. Follows SDLC chapters. |

## Quick orientation

- **Starting fresh?** Read `FEATURES.md` for a full picture of what the system does.
- **About to build something?** Check `TASKS.md` for the next task and how deep to go.
- **Looking for a file?** Check `CODEBASE.md`.
- **Setting up the infrastructure?** Follow `SETUP.md`.
- **Need deep design detail?** Check `DESIGN_SPECS.md`.
- **Want to understand how decisions were made?** Read `JOURNEY.md`.
